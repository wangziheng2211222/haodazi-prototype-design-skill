# 蝉圈圈交互蓝图

## 适用范围

当任务提到蝉圈圈、圈圈、达人、样品、合作、团长、店铺商品、内容中心、指挥中台、系统配置等业务时使用。

使用本蓝图回答：

- 功能应该归属哪个模块。
- 入口应该放在左侧导航、顶部 Tab、右侧列表操作区、筛选区、卡片操作区还是弹窗/抽屉。
- 页面之间是同级导航、Tab 切换、弹窗/抽屉还是返回跳转。
- 新增、导出、批量操作、配置等按钮应靠近哪个业务对象。

## 边界

- 本蓝图只覆盖已采集范围。
- “带货”“蝉圈圈操作演示”等未点击入口不视为已有能力。
- 未采集的弹窗/抽屉内容只能标记为待确认，不要编造成确定结构。
- 当前需求优先；若需求明确改变入口位置，记录偏离原因。

## 全局导航模式

- 多数页面为同级导航切换，不是深层父子页面。
- 常见返回方式是点击导航栏其他 Tab；弹窗/抽屉通过关闭返回原状态。
- 平台维度常以 Tab 呈现：全部、抖音、快手、视频号、小红书。
- 页面内主操作通常位于右侧列表操作区或筛选区域附近。
- 配置、导出、批量操作靠近当前列表或统计区域，不放到全局导航。

## 常用落位规则

- 和“达人发现、榜单、推荐、订阅品牌、品牌/商品找达人”相关，优先落在找达人模块。
- 和“已合作达人、出单达人、公海、标签、白名单/黑名单、矩阵号、跟进记录”相关，优先落在达人管理模块。
- 和“寄样、申样、样品订单、收货信息”相关，优先落在样品管理模块。
- 和“合作单、媒介合作、排期、达播”相关，优先落在合作管理模块。
- 和“团长合作、团长成交、团长转移、添加团长合作”相关，优先落在团长管理模块。
- 和“店铺、商品、商品库、商品分类、推品记录、店铺分组”相关，优先落在店铺商品模块。
- 和“品牌视频、行业热点、AI 拉片、脚本、仿写、内容资产、投放”相关，优先落在内容中心。
- 和“经营大盘、榜单、达播分析、商品分析、团队业绩、内容分析、大屏”相关，优先落在指挥中台。
- 和“角色、员工、部门、业务规则、权限、通知、监控、日志、账号信息”相关，优先落在系统配置。

## 页面结构线索

- 店铺分组页：左侧导航“店铺商品” -> “店铺管理” -> 顶部 Tab“店铺分组”；主区域是编辑分组区域；主操作是新增店铺分组。
- 找达人页：左侧导航“找达人” -> “智能推荐达人”；主区域为筛选区域和推荐达人列表；主操作是调整推荐策略、批量加入邀约池。
- 达人管理页：左侧导航“达人管理” -> “出单达人”；主区域为统计区域和达人列表；筛选包含员工、店铺；表格字段包括达人信息、员工、成交金额、退款金额、佣金、订单数、销量、寄样数、标签等。
- 团长管理页：左侧导航“数据工作台”或“团长管理” -> “团长管理”；主区域为统计区域和团长列表；筛选包含店铺；表格字段包括团长名称、合作媒介、归属模式、成交金额、计佣金额、出单达人数、出单商品数、退款金额、服务费、内容数、跟进记录、备注、操作等。
- 团长管理存在双来源线索：主蓝图将其列为“团长管理”模块，详情采集入口显示为“数据工作台”下的团长管理。生成时优先保留当前页面上下文，不要把它固定成唯一顶级入口。

## 生成规则

- 生成蝉圈圈相关原型时，先用本蓝图决定页面归属和入口位置，再用蝉圈圈设计规范决定视觉。
- `interaction-blueprint.json` 应记录使用了本蓝图，并说明用它确定了哪些模块、入口或关系。
- 对蓝图未采集的按钮结果，只能生成合理弹窗/抽屉假设，并标记为 `assumed` 或待确认。
- 不要为了让页面更丰富而新增蓝图没有依据且需求也未提到的核心模块。
- 如果只做一个新功能，优先将它放入最接近的已有模块，而不是新增顶级导航。
- 如果新功能跨多个模块，创建主入口时仍选择一个主模块，并在其他模块放辅助入口或关联跳转。
